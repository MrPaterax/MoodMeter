MoodMeter

Project MoodMeter is an all-in-one integrated health system that acquires the user's vitals and displays the results in the MoodMeter app.

The MoodmMeter's sensors record heartrate, blood oximetry, body temperature, and skin resistance data. As the data is collected, it is dispatched to a PyMongo database and plotted using the Matplotlib library. The MoodMeter app requests the recently plotted data and displays it in a easy to read view to the user. From the readings of the graph the app can detect shifts in the user's mood and alerts the user if they are feeling anxious.

Motivation

Motivation

After our pre-project labs were over we were assigned to create an IoT device. With so many IoT devices out on the market, it was difficult to propose a novel idea.

Attending Columbia University has proven to be quite a stressful journey, to say the least. With multitudes of projects, exams, and homework assignments due, a student can begin to feel anxious. We see this firsthand not only in ourselves but in our peers as well.

With this in mind, we wanted to develop an IoT device that would measure the vitals of a user and deduce how they were feeling. Alongside the device the integrated app would deduce and notify the user of possible feelings of anxiety.

System

Architecture

At the center of our block diagram is the brain of our system architecture, the Huzzah Feather ESP 8266. Connected to the ESPs I2C and Analog to Digital converter pins are the temperature sensor, heart rate and blood oximetry sensor, and lastly the Galvanic Skin Response sensor (GSR). The ESP sends the data from the sensors to the Amazon Web Services (AWS) database. The MoodMeter App requests the data from the server. The graphs are displayed on the app letting the user easily see the current status.

Hardware Components

• Huzzah Feather ESP 8226

As can be seen, we were only able to implement the four graphs of each of the sensor's data readings. For most of the project's development phase, we struggled with sending the data to the app in an easy-to-plot format. As a result, we decided to plot the readings using the Matlib library and display those plots in the app. Our hardware stayed the same except for the BlueTooth module since we decided to implement an HTTP GET request to acquire the data.
